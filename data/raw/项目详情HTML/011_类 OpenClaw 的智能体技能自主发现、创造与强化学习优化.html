任务描述
OpenClaw 的爆发证明了 Skill 生态的巨大价值——ClawHub 已托管 13,700+ 社区技能，Agent 甚至能自主识别能力缺口并生成新 Skill。但当前自创 Skill 质量参差不齐，缺乏系统性的质量保证和持续优化机制。衔远科技在企业 Agent 落地中发现，真实业务工具需求极度长尾——标准 API 仅覆盖约 30% 的操作。本课题探索让 Agent 像高级程序员一样自主发现需求→创造工具→测试验证→通过 RL 持续优化技能质量的完整闭环，是 Agent 从「被动执行」到「主动能力拓展」的关键跃迁。
1）实现 Skill Creation Agent：根据任务失败自动识别能力缺口，编写、测试、注册新 Skill 2）构建 Skill 质量评估与 RL 优化机制：通过使用反馈自动迭代提升 Skill 的正确性、效率和泛化性 3）在衔远 SKILL.md 框架下实现技能版本化管理与跨 Agent 共享
1. 能力缺口自动检测：Agent 如何判断现有 Skill 不足，需要创造新工具 2. 代码正确性保证：自创 Skill 的单元测试自动生成与沙箱验证 3. Skill 泛化性：避免过于特化的一次性工具，通过 RL 信号提升跨任务复用率 4. 安全沙箱：自主代码生成的执行隔离与权限控制
1. 衔远平台提供工具注册、调用和管理 API 2. SKILL.md 技能框架与 DeepAgents-CLI 工具链 3. 企业真实长尾工具需求案例库 4. 代码执行沙箱环境
第1周：体验 OpenClaw Skill 创建，分析真实任务中的长尾工具需求 第2周：设计 Skill Creation Agent 架构与 Skill 质量评分体系 第3周：实现「需求识别→代码生成→测试→注册」完整闭环 第4周：引入 RL 优化循环，通过使用反馈迭代 Skill 质量 第5周：在真实业务场景验证 第6周：整理技能库、撰写报告
