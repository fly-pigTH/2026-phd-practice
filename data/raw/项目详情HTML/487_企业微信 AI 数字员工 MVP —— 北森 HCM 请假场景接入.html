任务描述
医渡科技内部使用多套业务系统（北森 HCM、泛微 OA、合思费控、工时系统等），员工需在不同系统间来回切换、填写重复表单，操作摩擦力极高。本课题旨在以企业微信作为唯一交互入口，通过 AI Agent（基于 OpenClaw 框架）实现自然语言驱动的后台系统操作，让员工用一句话就能完成请假申请、费用报销、工时填报等日常事务，实现“人适应系统”到“系统服务人”的范式转变。考虑到系统集成复杂度，本课题选取单一系统的核心功能作为切入点，验证方案可行性后再逐步扩展。
1. 交付可运行的企业微信 AI 请假助手 Demo（支持发起、查询、撤销请假）； 2. 完成北森 HCM 请假场景的端到端自动化验证，错误率 < 5%； 3. 输出系统接入 SOP 文档，可指导后续更多 HCM 功能的快速接入。
1. 自然语言解析：如何从“我下周三下午要请半天假”这类口语化表达中准确提取结构化参数（类型、起止时间）； 2. 接口适配：北森 HCM 若无完整开放 API，如何通过 GUI 自动化或 Session 注入方式实现非侵入式操作； 3. 交互设计：多轮对话中如何处理信息缺失、歧义确认、审批状态异步回调等复杂状态。
1. 提供北森 HCM 系统测试账号及接口对接文档； 2. 提供企业微信应用开发权限（含自建应用 AgentID 及 API Key）； 3. 提供 OpenClaw 本地开发环境及技术文档； 4. 提供开发测试沙箱，数据不出内网； 5. 配备数字化运营部负责人及后端工程师联合指导。
第 1 周：调研与环境搭建 - 调研北森 HCM 接口文档，评估可用 API 范围 - 搭建企业微信开发环境，跑通消息收发基础流程 - 完成 OpenClaw 本地环境配置与 Hello World 验证 第 2-3 周：核心功能开发 - 开发北森 HCM 请假适配器（API 调用或 GUI 自动化） - 实现意图识别与参数提取（基于 LLM Function Calling） - 完成企业微信消息交互与确认流程 第 4-5 周：集成联调与测试 - 完成端到端流程联调（提交-审批-通知） - 编写边界测试用例，覆盖歧义、异常场景 - 优化交互体验，降低错误率至 5% 以下 第 6 周：交付与文档 - 录制 Demo 演示视频 - 完善 SOP 接入文档 - 撰写项目报告，进行成果汇报
