任务描述
随着先进工艺节点下 ASIC 芯片规模持续攀升，千万门级、亿门级芯片已成为行业常态，子系统级大模块 RTL 设计（如处理器核、高速接口 IP、复杂算法加速模块）已成为芯片研发的核心环节，占据了研发周期中大量的人力与时间成本。 当前 LLM 在 RTL 代码生成领域已实现初步落地，但仅能稳定支持计数器、FIFO、简单状态机等小规模模块的端到端生成；面对功能复杂、接口繁多、时序约束严格的大模块设计时，普遍存在功能遗漏、接口冲突、时序不收敛、可综合性差、长逻辑推理错误等核心问题，端到端生成的工程可用性几乎为零。 行业实践与学术研究均表明，符合 ASIC 自顶向下设计规范的层级化任务拆解，是突破 LLM 大模块 RTL 生成瓶颈的核心路径。但当前行业内尚无成熟的自动化拆解方案：人工拆解依赖资深工程师经验，耗时久、门槛高，无法发挥 LLM 的效率优势；通用 LLM 任务拆解方法不匹配硬件设计的工程规范，无法保证拆解后子模块的接口一致性、功能完整性与可集成性，拆而不合、拆而不可用的问题突出。
构建一套适配 ASIC 自顶向下设计规范的 LLM 自动化大模块任务拆解框架，实现大模块 RTL 设计需求的自动化、规范化层级拆解，解决 LLM 大模块 RTL 生成的核心痛点，支撑 LLM 完成高可用、高可综合性的大模块 RTL 代码生成。
1） 大模块 RTL 设计需求的结构化解析技术 解决自然语言 / 规格文档形式的大模块设计需求，无法被 LLM 精准理解、无法匹配硬件设计规范的问题，实现非结构化需求到结构化、层级化设计需求的自动化转化。 2）适配 ASIC 设计规范的层级化拆解规则体系构建。 解决通用 LLM 任务拆解不匹配硬件设计特性的核心痛点，构建符合自顶向下设计流程、覆盖功能边界、接口、时钟域、依赖关系、时序约束的硬件专属拆解规则库。 3）面向 RTL 大模块的自动化拆解 Agent 研发 基于 CoT/ToT 等推理技术，研发适配硬件设计的拆解 Agent，实现大模块的自动化层级拆解、子模块功能定义、接口自动生成、依赖关系全链路管控，替代人工拆解环节。
1）算力与开发资源：配备高性能 GPU 算力资源，提供主流商用大模型 API 完整授权，支持开源大模型的部署与推理，完全满足 Agent 开发、测试与全流程验证的算力需求； 2）EDA 工具与验证环境：拥有 Synopsys VCS、Design Compiler 等全流程商用 EDA 工具正式授权，覆盖 RTL 仿真、逻辑综合、功能覆盖率统计等全环节，提供完善的验证环境与标准化测试用例集； 3）技术与数据积累：已完成 LLM 辅助 RTL 设计的前期预研，积累了海量工程级大模块 RTL 设计数据集（含 RISC-V 核、高速接口、算法加速模块等）、商用设计规范、总线协议 IP 与验证平台，已构建基础的 Prompt 框架与 LLM-RTL 生成原型，为项目开展奠定坚实基础；
1. 第 1 周：岗前培训与技术方案设计 核心内容：完成项目背景、技术栈、EDA 工具、算力平台的岗前培训与实操演练；完成领域前沿论文深度调研与竞品分析；输出全流程技术方案、详细实施计划、里程碑节点与验证标准，通过项目组评审。 阶段输出：技术方案文档、实施计划表、前沿调研笔记 2.第2-5周：进行具体开发任务 3. 第 6 周：成果固化与结题验收 核心内容：完成项目全流程技术文档、工具使用手册、专利交底书的撰写；完成代码、数据、文档等项目资产的归档；完成项目成果汇报与专家评审，完成实习结题验收。 阶段输出：结题报告、完整技术文档、项目归档资产、成果汇报 PPT
