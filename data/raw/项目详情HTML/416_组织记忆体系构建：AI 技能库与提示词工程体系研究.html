任务描述
随着医渡科技向 AI Native 型组织转型，如何将员工的工作行为、业务知识与最佳实践沉淀为可复用的组织记忆，成为核心命题。当前公司各业务条线（GCP 交付、数字临床、BPO 等）存在知识碎片化、经验无法跨团队流转等痛点，亟需构建系统性的知识管理与技能中台体系，实现从“人工经验”到“机构记忆”的跨越，支撑全公司 64 个项目的高效交付。
1. 输出 AI 技能库架构设计文档（含分类体系、封装规范、版本管理方案）； 2. 完成 10 个以上高频业务场景的提示词模板封装并入库； 3. 建立技能效果评估框架，完成至少 3 个技能的基准测试。
1. 如何设计技能封装标准，使 Prompt 既能被人理解、又能被机器直接调用； 2. 如何建立跨部门的技能贡献与审核机制，保证质量； 3. 如何设计可量化的技能评估指标，覆盖效果、安全、成本三个维度。
1. 公司已有多云 LLM API 接入（AWS Bedrock / Azure OpenAI / 阿里云百炼），可提供开发测试额度； 2. 提供与各业务线 AI 使用骨干员工的访谈机会； 3. 提供内部已有的 Prompt 模板资产作为参考基础； 4. 配备数字化运营部负责人及 AI 工程师联合指导。
第 1-2 周：调研与需求梳理 - 访谈各业务线 AI 使用骨干，收集高频场景与痛点 - 调研 3-5 个主流 AI 技能平台架构，完成对比分析 - 梳理公司现有 Prompt 资产，评估质量与可复用性 第 3-4 周：体系设计与封装 - 设计技能分类体系与封装标准规范 - 完成 10 个以上高频场景的 Prompt 模板封装 - 搭建技能库原型（基于 Dify 或 Notion） 第 5-6 周：评估验证与交付 - 建立评估框架，完成基准测试 - 收集内测反馈，迭代优化 - 撰写技能库建设报告，进行成果汇报
