任务描述
在当前的工业制造与安防监控领域，工业AI视觉正面临着前所未有的复杂挑战。一方面，传统监督检测模型在面对诸如3C高速组装线等极速场景时，必须不断追求极致的推理速度 。另一方面，随着多SKU小批量生产模式的普及，企业频繁换型，传统需要大量数据标注的单一监督模型已难以应对高昂的标注成本和低频、长尾缺陷的泛化需求。 此外，在复杂的工业动态监控场景中，受限于光线、背景、视角等因素，单一目标检测模型极易出现精度不足与误判，难以精准评估复杂的动态行为轨迹与合规状态。为了满足工业界“换型不掉泪、缺陷不漏检、部署不求人”的迫切诉求 ，行业技术路线正明显向“高速骨干主检 + 大模型兜底”的双引擎架构演进。因此，亟需研发一套打破单一监督模型局限、融合多模型协同、支持极速换型且具备自然语言对话式检索能力的工业视觉模型推训与分析平台，为复杂的工业生产与监控场景提供系统级的视觉基座支持。
本项目旨在打造一套高可用、高拓展的综合性机器视觉PaaS平台，彻底改变传统“烟囱式”的视觉定制开发模式。具体建设目标分为以下四个核心维度： 目标一：构建“骨干主检 + 大模型兜底”的双引擎高并发架构。针对高速稳定产线与复杂换型场景的混合需求，底层集成YOLOv12、YOLO26及RF-DETR等极致实时检测骨干以保障处理节拍 。同时，上层无缝接入Qwen-VL、InternVL等多模态视觉大模型（VLM）及SAM3工业变体，对复杂或长尾异常进行二次确认和像素级兜底，实现速度与泛化能力的最佳平衡，显著降低系统的整体漏检率。 目标二：实现零/少样本极速换型流水线。针对缺陷种类繁多、标注成本高昂的痛点，引入基于CLIP、DINOv2等的工业微调范式 。赋能平台仅需输入少量参考图与文字描述即可精确定位新缺陷，实现“五分钟极速换型”，并在面对新工况和新纹理时，保证部分场景下准确率高达95%以上甚至99%，彻底摆脱对全监督海量数据标注的依赖。 目标三：研发多模型联合推理与复杂动态场景决策机制。摒弃单一依赖检测框的评判标准，联合串联目标检测、姿态估计及多目标跟踪等多维模型。针对复杂的工业连续动作（如操作规范、奔跑防误判等），引入“连续高置信度帧校验”机制过滤误判，并通过“置信度偏向值处理（如取单一画面特征极值或80%效果）”解决光线视角带来的状态或数量统计偏差，最终将多模型结果结合关键截图交由大模型进行综合裁决。 目标四：搭建自适应视频流处理与自然语言事件检索中枢。开发支持RTSP协议的24小时多线程视频拉流基座，结合跳帧模糊筛选与“滑动窗口+固定时长”算法，自适应提取包含完整交互上下文的高价值视频片段。基于Docker进行私有化部署，将多模态视觉特征输入Milvus向量数据库，并深度融合DeepSeek等文本大模型，实现最终系统能够通过自然语言指令精准追溯、检索出特定时间与人员的具体行为记录。
1、极速生产节拍与长尾/新发缺陷高泛化检测之间的架构融合问题 在工业场景中，诸如3C高速组装线要求极高的检测速度，而传统单靠YOLO等监督模型在面对低频或长尾缺陷时容易出现漏检。需要解决如何构建“骨干检测+大模型兜底”的双引擎混合架构，将要求极致速度的传统检测模型（如YOLOv12、RF-DETR）与多模态大模型（如Qwen-VL、SAM3工业变体）进行高效的底层串联与并行计算，在保证推理延迟的前提下，将整体漏检率压降至0.3‰以下。 2、多SKU小批量生产场景下的高昂标注成本与极速换型难题 针对一月换型10次以上或缺陷种类超过50种的复杂生产模式，传统的全监督模型训练方式面临极高的数据收集和标注成本。需要解决如何引入并优化Zero/Few-Shot（零/少样本）学习范式，利用CLIP、DINOv2等预训练模型进行工业微调，使得系统能够仅依赖少量正常样本与文字描述（或几张参考图）即可精准定位新纹理与新缺陷，彻底攻克“五分钟极速换型”的技术瓶颈。 3、复杂动态场景下单帧视觉模型的精度衰减与动作轨迹误判问题 受限于光线、背景、视角等多方位因素，单一目标检测模型（如YOLO）在统计人物数量或判断复杂动作时极易出现精度不够高的问题。需要重点攻克多模型融合与时序校验算法：一是针对数量与状态评估，研究“偏向值处理”算法（如取画面特征偏向80%的效果或最大极值）以消除单帧波动；二是针对动作判断（如因转身瞬间轨迹过大被误判为奔跑），研究基于多目标跟踪（ByteTracker）与姿态估计（YOLO-Pose）的“连续高置信度帧校验”机制，并将多模型结果集与截取画面交由大模型进行最终的精确裁决。 4、海量高并发视频流的上下文自适应截取与跨模态自然语言检索问题 在24小时不间断的视频流（RTSP协议）处理中，如何高效剥离冗余画面并实现智能化业务追溯是一个巨大的工程挑战。需要解决两个核心难点：首先是底层视频处理流转，研究如何结合跳帧模糊筛选与“滑动窗口+固定时长”算法，以最低的算力成本截取保留完整动作交互上下文的视频片段；其次是跨模态语义映射，研究如何将视觉结构化数据与大模型判定结论存入Milvus等向量数据库，并打通与DeepSeek等文本推理大模型的语义通道，最终实现仅通过自然语言沟通即可精准检索特定时间与人员的具体事件片段 。

第1周：基础环境搭建与底层视频/推理基座构建 视频流与后端： 搭建基于 Docker 的私有化部署环境；开发自适应视频流处理基座，实现基于 RTSP 协议的多线程 24 小时拉流架构，并完成跳帧模糊筛选逻辑的开发。 视觉算法： 完成极致实时检测骨干（如 YOLOv12、YOLO26、RF-DETR 等）的选型与基础部署，搭建基于 TensorRT 或 NCNN 的底层硬件加速计算环境。 第2周：零/少样本微调流水线与大模型兜底接入 视觉算法： 引入 CLIP、DINOv2 等预训练模型，研发面向复杂工业场景的零/少样本（Zero/Few-Shot）微调机制；接入 Qwen-VL、InternVL 等多模态视觉大模型及 SAM3 工业变体，实现像素级异常分割与漏检兜底功能的初步走通。 平台工程： 建立“五分钟换型”业务逻辑，即仅依赖少量参考图或文字描述即可完成新工况、新缺陷的特征库匹配与热更新。 第3周：多维模型协同推理与动态防误判机制开发 视觉算法： 联合串联目标检测（YOLO）、姿态估计（YOLO-Pose）及多目标跟踪（ByteTracker）管线。 认知与策略： 针对动态复杂场景开发特定的校验算法：实现“连续高置信度帧校验”以解决转身等瞬间动作造成的轨迹误判；开发“偏向值处理”算法（如取画面极值或偏向80%的效果），解决因光线、背景导致的单帧数量或状态检测精度波动问题。 第4周：视频智能切分与自然语言向量检索打通 视频流与后端： 研发“滑动窗口+固定时长”的视频片段切分算法，确保截取的视频段能够最大程度保留完整的动作交互上下文。 检索与大模型： 搭建 Milvus 向量数据库存储多模态结构化数据；接入 DeepSeek 等文本推理大模型，打通“自然语言指令 -> 文本特征 -> 视频向量映射 -> 事件片段检索”的底层核心链路。 第5周：双引擎架构闭环与系统级高并发压测 系统集成： 彻底打通“骨干主检 + 大模型兜底”的双引擎混合调度架构，将前期抽取的多模型结果集与核心关键截图打包，交由大模型进行最终的精准裁决。 测试：在模拟的高速产线或复杂多SKU换型场景下进行高并发压力测试，以期达到将漏检率压降至 0.3‰ 以下的目标。 第6周：前后端交互联调、演示验收与项目交付 平台工程： 完善基于前端框架（如 React 等）与后端（如 FastAPI）的推训平台可视化界面联调 ，沉淀系统功能与 API 接口文档。 项目交付： 梳理测试报告，准备“极速产线缺陷兜底”、“五分钟极速换型无缝热更”以及“自然语言对话式视频溯源”等核心演示用例，向相关方全面展示平台的综合能力。
