任务描述
医渡科技内部使用多套业务系统（北森 HCM、泛微 OA、合思费控、工时系统等），员工需在不同系统间来回切换、填写重复表单，操作摩擦力极高。本课题旨在以企业微信作为唯一交互入口，通过 AI Agent（基于 OpenClaw 框架）实现自然语言驱动的后台系统操作，让员工用一句话就能完成请假申请、费用报销、工时填报等日常事务，实现“人适应系统”到“系统服务人”的范式转变。考虑到系统集成复杂度，本课题选取单一系统的核心功能作为切入点，验证方案可行性后再逐步扩展。
1. 交付可运行的企业微信 AI 审批助手 Demo（支持发起申请、查询进度、催办）； 2. 完成泛微 OA 至少 2 个核心审批场景的端到端自动化验证； 3. 输出安全权限设计方案与系统接入 SOP 文档。
1. 接口适配挑战：泛微 OA 部分功能无标准 API，需通过 GUI 自动化或逆向分析 Web 接口实现非侵入式对接； 2. 权限安全：如何设计细粒度权限校验，防止 Agent 被用于提交超出用户权限范围的审批； 3. 流程多样性：不同审批类型的表单字段差异大，如何设计通用的表单理解与填写框架。
1. 提供泛微 OA 系统测试账号及现有接口文档； 2. 提供企业微信应用开发权限； 3. 提供现有审批流程的截图和操作手册，用于 GUI 自动化分析； 4. 提供 OpenClaw 本地开发环境及 Agent 开发测试沙箱； 5. 配备数字化运营部负责人及 OA 系统管理员联合指导。
第 1 周：调研与环境搭建 - 调研泛微 OA API 文档，评估 REST API 覆盖范围 - 分析 2 个核心审批场景的表单结构与流程逻辑 - 搭建企业微信开发环境，完成 OpenClaw 配置 第 2-3 周：核心功能开发 - 开发泛微 OA 接口适配器（优先 API，备用 GUI 自动化） - 实现审批发起、字段填写与提交的 Agent 技能封装 - 设计并实现权限校验机制 第 4-5 周：集成联调与测试 - 完成 2 个场景的端到端联调 - 测试权限边界场景，验证安全机制有效性 - 完成审批状态查询与催办功能 第 6 周：交付与文档 - 录制 Demo 演示视频 - 完善 SOP 接入文档与安全方案报告 - 撰写项目报告，进行成果汇报
